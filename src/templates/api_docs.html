API Reference

Programmatic access to repo ingestion, AI-powered content generation, and usage analytics.

BASE URL
https://repolm.com/api/v1

Authentication

All API requests require an X-API-Key header. Generate your key from Account Settings → API Key (click your avatar in the app).

Example request
curl https://repolm.com/api/v1/usage \
  -H "X-API-Key: rlm_sk_abc123def456"

⚠️ Keep your API key secret. Do not expose it in client-side code or public repositories.

Quick Start

One request. One response. That's it.

Analyze a repo in one request
curl -X POST https://repolm.com/api/v1/analyze \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://github.com/fastapi/fastapi"}'
pip install requests
import requests

resp = requests.post("https://repolm.com/api/v1/analyze",
    headers={"X-API-Key": "YOUR_API_KEY"},
    json={
        "url": "https://github.com/fastapi/fastapi",
        "kind": "overview",
        "depth": "high-level",
        "expertise": "intermediate"
    })

print(resp.json()["content"])

⏲ The request blocks until the result is ready (typically 30–90 seconds). No polling needed.

Endpoints

Analyze a GitHub repo and get the result in a single request. Handles ingestion + generation automatically. Blocks until complete.

Request Body
JSON
{
  "url": "https://github.com/fastapi/fastapi",

  "depth": "high-level",
  "expertise": "intermediate"

FieldTypeDefaultDescription

depthstring"high-level"high-level or in-depth
expertisestring"amateur"amateur, intermediate, or expert
Response
200 OK
{
  "url": "https://github.com/fastapi/fastapi",
  "kind": "overview",
  "depth": "high-level",
  "expertise": "intermediate",
  "content": "# FastAPI Overview\n\nFastAPI is a modern, high-performance...",
  "token_cost": 4
}
Token costs by kind
overview4 tokens (1 ingest + 3 generate)
slides6 tokens (1 ingest + 5 generate)
podcast9 tokens (1 ingest + 8 generate)

Advanced: Async Workflow

For more control (webhooks, progress tracking, reusing ingested repos), use the individual endpoints below. The flow is: ingest → poll → generate → poll → result.

Endpoints

POST /repos

Ingest a GitHub repository. The repo will be cloned, parsed, and indexed for content generation.

Request Body
JSON
{
  "url": "https://github.com/fastapi/fastapi"
}
FieldTypeRequiredDescription
urlstringYesGitHub repository URL (or owner/repo shorthand)
Response
200 OK
{
  "repo_id": "a1b2c3d4",
  "token_cost": 1,
  "queued": true,
  "queue_position": 3
}
Response fields
repo_idstringUnique identifier for this ingestion
token_costintegerTokens spent for this operation
queuedbooleanPresent if request was queued (server busy)
queue_positionintegerPosition in queue (only if queued)
GET /repos/{repo_id}

Check the status of a repo ingestion. Poll this endpoint until status is "ready".

Request
curl https://repolm.com/api/v1/repos/a1b2c3d4 \
  -H "X-API-Key: rlm_sk_abc123def456"
Response
200 OK
{
  "status": "ready",
  "message": "Ingestion complete",
  "data": {
    "name": "fastapi",
    "language": "Python",
    "stars": 75000
  },
  "file_count": 342
}
Status values
queuedWaiting in queue to be processed
processingCurrently being ingested
readyIngestion complete — you can now generate content
errorSomething went wrong (check message)

GET /jobs/{job_id}

Poll the status of a generation job. When status is "done", the result field contains your generated content.

Request
curl https://repolm.com/api/v1/jobs/x7y8z9w0 \
  -H "X-API-Key: rlm_sk_abc123def456"
Response (in progress)

{
  "status": "processing",
  "message": "Generating overview...",

{

  "message": "Complete",

Error Reference

All errors return JSON with an error field.

Error response format
{ "error": "Description of what went wrong" }
StatusMeaningHow to Fix
400 Bad request — missing or invalid parameters Check your request body. The url field is required for ingestion; repo must be "ready" for generation.
401 Unauthorized — missing or invalid API key Include a valid X-API-Key header. Generate one from Account Settings → API Key.
402 Insufficient tokens Your token balance is too low. Purchase more tokens or wait for your free tier to reset. Response includes required and balance fields.
404 Not found The repo or job ID doesn't exist. Double-check the ID from the creation response.
429 Rate limit exceeded You've hit your daily API call limit. Upgrade your plan or wait until tomorrow. Response includes your limit.
503 Server busy All processing slots are full. Retry after a few seconds.
